# 프로젝트 개요
현대 암호 체계의 근간인 RSA 알고리즘을 직접 구현하며, 데이터의 기밀성과 무결성을 보장하는 프로세스를 학습했습니다. 
초기 6바이트 키 생성 모델에서 시작하여, 현재는 보안 표준인 **2048-bit** 키 생성 및 **Base64 기반 전송 시스템**으로 확장되었습니다.

## RSA 알고리즘
두 개의 큰 소수를 기반으로 공개키/개인키를 생성하는 **비대칭 암호화** 방식입니다.

### 키 생성 과정
1. **N 계산**  
   $N = p \times q\$, ($p,\ q$는 서로 다른 큰 소수)

2. **φ(N) 계산**  
   $\varphi(N) = (p-1)(q-1)$

3. **암호화 지수(e) 선택**  
   - 조건 1: $1 < e < \varphi(N)$  
   - 조건 2: $\gcd(e,\ \varphi(N)) = 1$ 즉, ($e와\ \varphi(N)$은 서로소)

4. **복호화 지수(d) 계산**  
   $(e \cdot d) \mod \varphi(N) \equiv 1$

### 암호화/복호화
| 과정       | 수식                     |
|------------|--------------------------|
| 암호화    | $C = M^e \mod N$      |
| 복호화    | $M = C^d \mod N$      |

# 주요 함수 설명
| 함수명 | 기능 | 수학적 원리 |
|--------|------|-------------|
| isPrime() | 확률적 소수 판별 | $n - 1 = 2^s \cdot d$ |
| phi() | 오일러 피 함수 | $\varphi(N) = (p-1)(q-1)$ |
| createExponent() | 유효한 e 검증 | $\gcd(e,\ \varphi(N)) = 1$ |
| extendEuclid() | 모듈러 역원 계산 | $(e \cdot d) \mod \varphi(N) \equiv 1$ |

# 상세 구현 설명
**1. 고성능 소수 생성 파이프라인**  
2048-bit의 거대 정수 환경에서 보안성을 갖춘 소수를 빠르게 생성하기 위해 **다단계 필터링 아키텍처**를 설계했습니다.
- **Step 1. 짝수 판별:** 가장 먼저 짝수를 즉시 제거합니다.
- **Step 2. 소규모 소수 전수 조사:** 1,000 이하의 모든 소수로 나누어 떨어지는지 확인합니다. 이 과정을 통해 밀러-라빈 연산이 필요한 후보군을 대폭 줄여 전체 연산 비용을 최적화했습니다.
- **Step 3. 밀러-라빈 판별 (Miller-Rabin Test):** $n - 1 = 2^s \cdot d$ 형태의 분해를 이용한 확률적 소수 판별을 수행합니다.
- **성능 지표:** 이 3단계 파이프라인을 통해 무작위 후보군 1,000개 검사 시 20초 이내라는 성능을 달성했습니다.

**2. 확장 유클리드 알고리즘을 통한 키 생성**  
라이브러리 도움 없이 **확장 유클리드 알고리즘**을 직접 구현하여 수학적 정밀도를 확보했습니다.
- **모듈러 역원 계산:** $e*d \equiv 1 \pmod{\varphi(N)}$을 만족하는 역원 $d$를 산출합니다.

**3. 암호화 지수(e) 선택 및 검증 로직**
- **자동 키 생성 (Auto):** 성능과 보안이 검증된 **65537($2^{16}+1$)** 을 기본값으로 사용합니다. (폴라드 로 공격 방어 및 연산 효율 극대화)
- **수동 키 생성 (Manual):** 사용자가 직접 입력한 지수에 대해 createExponent() 함수가 $\gcd(e, \varphi(N)) = 1$ 여부를 검사합니다. 특히 $\varphi(N) < 65537$인 특수 환경에서도 유효한 지수를 동적으로 식별하도록 설계되었습니다.

**4. Base64 기반 데이터 파이프라인**
- **Flow**: `String` ➔ `Byte` ➔ `BigInt` ➔ `RSA 연산` ➔ `Byte` ➔ `Base64`
- 암호화된 거대 정수를 텍스트 기반 환경에서도 데이터 유실 없이 안전하게 전송할 수 있도록 Base64 인코딩/디코딩 레이어를 구축했습니다.
